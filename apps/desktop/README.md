# @cursor-gateway/desktop

Windows desktop client — a **Tauri v2 + WebView2** shell that bundles and
**locally loads** the Secure Web E2EE UI (`apps/secure-web/dist`). It is the
recommended, MITM-resistant way to reach the encrypted-chat gateway on Windows.

This is **not** the browser extension and **not** a plain web PWA.

**Cloudflare Access (desktop):** click the key icon in the top-right after setting the
Gateway origin — the shell opens a same-site Access bridge window (local
`tauri.localhost` cannot send Access cookies cross-site). See
[`docs/windows-client.md`](../../docs/windows-client.md).

**Upgrade:** when a newer installer is published (`/api/desktop/version`), an
arrow-up icon appears; click to download and run the NSIS setup.

## Build (native Windows or CI — WSL/Linux cannot cross-compile)

```powershell
npm install
npm run build -w @cursor-gateway/secure-web
npm run build -w @cursor-gateway/desktop   # build:frontend + icon + tauri build (NSIS .exe)
```

Output: `src-tauri/target/release/bundle/nsis/*-setup.exe`.

Prereqs: Node 22+, Rust (stable, `x86_64-pc-windows-msvc`), WebView2 Runtime.
CI: [`.github/workflows/desktop-windows.yml`](../../.github/workflows/desktop-windows.yml).

Full build / download / install / verify guide: [`docs/windows-client.md`](../../docs/windows-client.md).

## Icon

`app-icon.png` (1024²) is committed and regenerated by `node scripts/gen-icon.mjs`
(pure Node, no image deps). `npm run icon` fans it out via `tauri icon`.
