# 快速开始（傻瓜式）

把下面三处里的占位符换成你自己的值即可。仓库里没有真实密钥。

| 占位符 | 含义 |
| --- | --- |
| `gateway.example.com` | 你的网关域名 |
| `<随机密钥>` | 自己生成的长随机串（≥32 字符） |
| `<Cursor API Key>` | Cursor 控制台里的 API Key |
| `C:\Workspaces\project1` | 允许 Runner 访问的本地目录 |

---

## 第一步：准备 VPS（约 10 分钟）

1. 安装 Docker（含 Compose 插件）。
2. 克隆本仓库到例如 `/opt/cursor-gateway`。
3. 复制环境文件并编辑：

```bash
cd /opt/cursor-gateway
cp .env.example .env
nano .env
```

4. **最少必改这些行：**

```bash
PUBLIC_ORIGIN=https://gateway.example.com
JWT_SECRET=<随机密钥A>
RUNNER_SHARED_SECRET=<随机密钥B>
POSTGRES_PASSWORD=<数据库密码>
DATABASE_URL=postgres://cursor_gateway:<数据库密码>@postgres:5432/cursor_gateway
ALLOWED_EMAILS=you@example.com
```

生成随机串示例：

```bash
openssl rand -hex 32
```

5. 启动：

```bash
cd /opt/cursor-gateway/infra
docker compose up -d --build
curl -fsS http://127.0.0.1:18080/healthz
```

看到健康检查成功即可。

6. 把现有 Nginx/Caddy/Cloudflare 反代指到本机 `127.0.0.1:18080`，并把 `infra/nginx-gateway.conf` / `infra/Caddyfile` 里的域名改成你的。  
   建议用 **Cloudflare Access** 保护整站（除 Runner 回调路径按你的策略配置）。

---

## 第二步：准备 Windows Runner（约 5 分钟）

1. 安装 [Node.js 22+](https://nodejs.org/)。
2. 在同一仓库根目录打开 PowerShell：

```powershell
.\apps\windows-runner\scripts\start-runner.ps1
```

3. 首次会生成 `apps\windows-runner\.env`，用记事本打开，填：

```text
GATEWAY_URL=https://gateway.example.com
RUNNER_SHARED_SECRET=<与 VPS 的 RUNNER_SHARED_SECRET 完全相同>
RUNNER_WORKSPACES=C:\Workspaces\project1
CURSOR_API_KEY=<Cursor API Key>
```

若站点前面有 Cloudflare Access，还需要 Service Token：

```text
CF_ACCESS_CLIENT_ID=<client id>
CF_ACCESS_CLIENT_SECRET=<client secret>
```

4. 再运行一次启动脚本。  
5. （推荐）管理员 PowerShell 安装开机自启：

```powershell
.\apps\windows-runner\scripts\install-runner-daemon.ps1
```

---

## 第三步：验证

1. 浏览器打开 `https://gateway.example.com`，用 Cloudflare Access 登录。
2. 页面应出现 Windows 上报的工作区与模型。
3. 选一个工作区，发一句只读问题，状态应变为 `finished`。

出问题先看：

- VPS：`docker compose -f infra/docker-compose.yml logs -f app`
- Windows：`apps\windows-runner\logs\` 下的日志与 `runner-health.json`

---

## 配置对照（避免填错）

| 配置项 | 填在哪里 | 两边要一致？ |
| --- | --- | --- |
| `PUBLIC_ORIGIN` / `GATEWAY_URL` | VPS `.env` / Windows `.env` | 域名应对应同一站点 |
| `RUNNER_SHARED_SECRET` | 两边都要 | **必须完全一致** |
| `JWT_SECRET` | 仅 VPS | — |
| `CURSOR_API_KEY` | 仅 Windows | — |
| `RUNNER_WORKSPACES` | 仅 Windows | — |
| `ALLOWED_EMAILS` | 仅 VPS | — |
| `TELEGRAM_*` | 仅 VPS（可选） | — |

---

## 不要做的事

- 不要把 `.env`、API Key、Cookie、Service Token 提交到 Git
- 不要把 Runner 工作区设成整个 `C:\Users`
- 不要对公网裸奔 18080 端口（应只监听本机并由反代/Access 保护）
